# TravelEngine â€“ Post-Clone Setup This guide covers everything you need to do **after cloning** the TravelEngine repository to get it running locally, including fixing the usual Laravel cache / `bootstrap/cache` issues. ## 1. Clone the repo ``` git clone travelengine cd travelengine ``` ## 2. Install PHP dependencies Make sure you have **PHP**, **Composer**, and the required PHP extensions (including `pdo_sqlite` if using SQLite). ``` composer install ``` ## 3. Create `.env` file and app key ``` cp .env.example .env php artisan key:generate ``` ## 4. Configure the database ### SQLite (simple local setup) ``` mkdir -p database touch database/database.sqlite ``` Update `.env`: ``` DB_CONNECTION=sqlite DB_DATABASE=database/database.sqlite ``` ### MySQL/Postgres Set the usual DB vars in `.env`. ## 5. Ensure `storage` and `bootstrap/cache` are writable Create missing dirs: ``` mkdir -p bootstrap/cache ``` Fix permissions (Windows example): ``` icacls ".\bootstrap\cache" /grant "Everyone:(F)" icacls ".\storage" /grant "Everyone:(F)" ``` ## 6. Run migrations ``` php artisan migrate ``` ## 7. Clear Laravel caches ``` php artisan config:clear php artisan cache:clear php artisan route:clear php artisan view:clear ``` Optionally rebuild: ``` php artisan optimize ``` ## 8. Install frontend deps ``` npm install npm run dev ``` ## 9. Run the application ``` php artisan serve ``` --- You can now start using TravelEngine locally.
